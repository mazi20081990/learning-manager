# 学习管家 - 剩余开发实施计划

## 概述

学习管家是一个基于AI的个性化学习系统，后端（Flask + SQLite）已完成约95%，前端仅有HTML骨架（约5%）。本计划聚焦于**完成前端三个缺失文件**和**修复后端已知bug**，使系统达到可运行状态。

**核心架构确认**：秘塔(MetaSo)仅用于搜索参考资料，大模型基于搜索结果整合生成实际学习内容。

## 实施步骤

### 步骤1：修复后端Bug

#### 1.1 修复 scheduler.py 导入问题
- **文件**: `backend/app/services/scheduler.py`
- **问题**: 第68行和第112行使用了 `get_db()`，但该函数在第187行才import，运行时会报 `NameError`
- **修复**: 将 `from app.models.database import get_db` 移到文件顶部（第7行之后）

#### 1.2 修复 database.py 默认管理员密码
- **文件**: `backend/app/models/database.py`
- **问题**: 第261行默认管理员密码为 `'sha256$admin123'`，这不是有效的 werkzeug 密码哈希格式，管理员无法登录
- **修复**: 使用 `generate_password_hash('admin123')` 的实际输出值替换。在init_db()函数开头添加 `from werkzeug.security import generate_password_hash`，然后将INSERT语句中的密码替换为 `generate_password_hash('admin123')` 的调用

### 步骤2：配置前端静态文件服务

#### 2.1 修改 Flask 应用配置
- **文件**: `backend/app/__init__.py`
- **修改**: 添加前端静态文件路由，使Flask能正确提供 `frontend/` 目录下的文件
- **方法**: 在Flask app初始化后，添加静态文件路由指向 `../frontend` 目录；或者更好的方式是添加一个catch-all路由来提供前端页面

## 假设与决策

1. **前端架构**: 采用纯原生HTML/CSS/JS，不引入React/Vue等框架，保持轻量级，适合NAS部署
2. **CSS方案**: 手写CSS + CSS变量，不使用Tailwind等框架，减少构建步骤
3. **API认证**: 使用Flask-Login的session认证，前端通过cookie自动携带
4. **静态文件服务**: Flask直接提供前端静态文件，无需额外nginx
5. **移动端优先**: CSS使用mobile-first策略，确保手机端体验优先
6. **离线能力**: PWA manifest提供基础离线支持，service worker暂不实现（后续可加）
