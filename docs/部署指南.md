# 学习管家 - 部署指南

## 系统要求

### 最低配置
- CPU: 1核
- 内存: 512MB
- 存储: 1GB
- 网络: 可访问互联网

### 推荐配置
- CPU: 2核
- 内存: 1GB
- 存储: 5GB
- 网络: 稳定的互联网连接

## 部署方式

### 方式一：直接部署（推荐开发环境）

#### 1. 环境准备
```bash
# 安装Python 3.11+
python --version

# 安装pip
pip --version
```

#### 2. 下载项目
```bash
git clone https://github.com/your-username/learning-manager.git
cd learning-manager
```

#### 3. 配置环境
```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
nano .env
```

#### 4. 安装依赖
```bash
pip install -r requirements.txt
```

#### 5. 启动应用
```bash
cd backend
python run.py
```

#### 6. 访问应用
打开浏览器访问 http://localhost:5000

### 方式二：Docker部署（推荐生产环境）

#### 1. 安装Docker
```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh

# CentOS/RHEL
sudo yum install -y docker

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker
```

#### 2. 下载项目
```bash
git clone https://github.com/your-username/learning-manager.git
cd learning-manager
```

#### 3. 配置环境
```bash
cp .env.example .env
# 编辑 .env 文件，填写必要的配置
```

#### 4. 构建镜像
```bash
docker build -f docker/Dockerfile -t learning-manager .
```

#### 5. 运行容器
```bash
docker run -d \
  --name learning-manager \
  -p 5000:5000 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/logs:/app/logs \
  -v $(pwd)/.env:/app/.env \
  --restart unless-stopped \
  learning-manager
```

#### 6. 查看日志
```bash
docker logs -f learning-manager
```

### 方式三：Docker Compose部署（推荐NAS环境）

#### 1. 安装Docker Compose
```bash
# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

#### 2. 下载项目
```bash
git clone https://github.com/your-username/learning-manager.git
cd learning-manager
```

#### 3. 配置环境
```bash
cp .env.example .env
# 编辑 .env 文件
```

#### 4. 启动服务
```bash
cd docker
docker-compose up -d
```

#### 5. 查看状态
```bash
docker-compose ps
docker-compose logs -f
```

#### 6. 停止服务
```bash
docker-compose down
```

## NAS部署（飞牛NAS）

### 1. 安装Docker
在飞牛NAS应用中心安装Docker套件

### 2. 上传项目文件
通过SMB/FTP将项目文件上传到NAS

### 3. 配置环境变量
在NAS上编辑.env文件

### 4. 创建Docker容器
```bash
# SSH连接到NAS
ssh admin@your-nas-ip

# 进入项目目录
cd /volume1/learning-manager

# 使用Docker Compose启动
cd docker
docker-compose up -d
```

### 5. 配置反向代理（可选）
在飞牛NAS控制面板中配置反向代理，将域名指向容器端口

### 6. 配置内网穿透
使用飞牛NAS自带的远程访问功能，或配置frp等内网穿透工具

## 配置详解

### 必需配置

#### Flask密钥
```env
FLASK_SECRET_KEY=your-secret-key-here
```
生成随机密钥：
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

#### 大模型API
```env
LLM_API_KEYS=sk-your-api-key-1,sk-your-api-key-2
LLM_API_BASE=https://api.siliconflow.cn/v1
LLM_MODEL=Qwen/Qwen2.5-7B-Instruct
```

支持的大模型平台：
- 硅基流动 (https://api.siliconflow.cn/v1)
- DeepSeek (https://api.deepseek.com/v1)
- 阿里云百炼 (https://dashscope.aliyuncs.com/compatible-mode/v1)

#### 秘塔搜索API
```env
MITA_API_KEY=your-mita-api-key
MITA_API_BASE=https://api.metaso.cn/v1
MITA_DAILY_LIMIT=80
```

获取方式：
1. 访问 https://metaso.cn
2. 注册账号
3. 在开发者中心获取API Key

#### 钉钉Webhook
```env
DINGTALK_WEBHOOK=https://oapi.dingtalk.com/robot/send?access_token=your-token
DINGTALK_SECRET=your-secret
```

获取方式：
1. 在钉钉群设置中添加机器人
2. 选择"自定义"机器人
3. 复制Webhook地址和加签密钥

## 定时任务配置

### 系统定时任务（推荐）
```bash
# 编辑crontab
crontab -e

# 添加以下任务
# 每日8:00生成学习内容
0 8 * * * cd /path/to/learning-manager/backend && python -c "from app.services.scheduler import generate_daily_content; generate_daily_content()"

# 每日8:30发送提醒
30 8 * * * cd /path/to/learning-manager/backend && python -c "from app.services.scheduler import send_daily_reminders; send_daily_reminders()"

# 每周日凌晨2:00备份
0 2 * * 0 cd /path/to/learning-manager/backend && python -c "from app.services.scheduler import weekly_backup; weekly_backup()"
```

### Docker环境定时任务
容器内已内置APScheduler，无需额外配置

## 数据备份

### 自动备份
系统每周自动备份SQLite数据库到 `data/backups/` 目录

### 手动备份
```bash
# 备份数据库
cp data/learning_manager.db data/learning_manager_backup_$(date +%Y%m%d).db

# 备份图片
tar -czf images_backup_$(date +%Y%m%d).tar.gz static/images/
```

### 恢复备份
```bash
# 停止应用
docker-compose down

# 恢复数据库
cp data/learning_manager_backup_20240101.db data/learning_manager.db

# 启动应用
docker-compose up -d
```

## 常见问题

### 1. 应用无法启动
检查环境变量是否配置正确，特别是：
- FLASK_SECRET_KEY
- LLM_API_KEYS
- MITA_API_KEY

### 2. 推送通知失败
检查Webhook配置：
- 钉钉Webhook地址是否正确
- 加签密钥是否配置
- 网络是否可以访问钉钉服务器

### 3. 搜索额度不足
- 检查秘塔API Key是否有效
- 查看额度使用情况
- 考虑减少每日搜索次数

### 4. 图片生成失败
- 检查NAS存储空间
- 检查网络连接
- 查看图片生成日志

### 5. 数据库锁定
SQLite不支持并发写入，如果出现锁定：
- 重启应用
- 检查是否有其他进程访问数据库

## 安全建议

1. **修改默认密码**：首次登录后修改管理员密码
2. **配置HTTPS**：生产环境使用HTTPS
3. **限制访问**：配置防火墙，仅允许必要IP访问
4. **定期备份**：定期备份数据库和重要数据
5. **更新依赖**：定期更新Python依赖包

## 性能优化

1. **使用CDN**：静态资源使用CDN加速
2. **启用缓存**：配置浏览器缓存策略
3. **数据库优化**：定期VACUUM SQLite数据库
4. **图片优化**：启用图片压缩和懒加载

## 升级指南

### 升级步骤
```bash
# 1. 备份数据
cp data/learning_manager.db data/learning_manager_backup.db

# 2. 拉取最新代码
git pull origin main

# 3. 更新依赖
pip install -r requirements.txt --upgrade

# 4. 重启应用
docker-compose restart
```

### 数据库迁移
升级时可能需要迁移数据库：
```bash
# 备份旧数据库
cp data/learning_manager.db data/learning_manager_old.db

# 重新初始化数据库（会清空数据，谨慎操作）
rm data/learning_manager.db
python -c "from app.models.database import init_db; init_db()"
```
